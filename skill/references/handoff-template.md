# 环节流转交接包模板

## 模板

```
## 🔄 流转：{项目代号} {当前环节} → {下一环节}

**结论：**
- 确认范围：{做了什么}
- 明确排除：{没做什么}

**交付物：**
- {链接/路径}

**验收标准：**
- {下一环节拿什么标准判断产出合格}

**已知风险/约束：**
- {上一环节发现的可能影响下一环节的问题}

**优先级：** Px
**期望完成时间：** YYYY-MM-DD

**反向触发条件：**
如遇 {xxx 情况}，@Pichai 讨论。
```

## PM 枢纽规则

**所有环节交付必须先回 PM：**
- 开发完成 → @PM → PM 检查 → PM 指派测试
- 测试发现 bug → @PM → PM 开修复任务 → PM 指派开发
- 修复完成 → @PM → PM 指派回归测试

**禁止：** 开发直接交付测试、测试直接找开发修。

## 流转后动作

1. 更新 Issues 的 Stage 字段
2. 确认下一环节负责人收到并理解交接包
3. 更新 GROUP.md 当前状态
