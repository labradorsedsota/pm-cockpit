# 任务指派七要素模板

## 模板

```
📌 任务指派 {任务ID}

1. 做什么：{具体任务描述}
2. 输入物：{链接/位置/注意事项}
3. 产出标准：{做到什么程度算完成}
4. 时间预期：{预计 X 小时，截止时间}
5. 执行模式：{直接执行 / 计划确认}
6. Issue 链接：{URL}
7. 请确认收到并告知是否可以开始
```

## 执行模式说明

| 模式 | 适用场景 | 接收方流程 |
|------|---------|-----------|
| 直接执行 | 简单明确的任务，无歧义 | ACK → 立即开始 |
| 计划确认 | 复杂/有歧义的任务 | ACK → 回复执行计划 → PM 确认 → 开始 |

由 PM 在指派时标注。

## 指派后动作

1. Issue Status → `Assigned`，更新 Status Since
2. 等 ACK → 收到后 Status → `In Progress`
3. **30 分钟未 ACK → 主动询问一次**
4. **仍无回应 → 预警**

## 测试任务附加要求

指派 Moss 执行测试时，额外嵌入：
- mano-cua 优先（选 browser API 须注明原因）
- 引用测试执行规范版本号
- Fixture 文件位置
- 预期结果逐条列出，禁止压缩
