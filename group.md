# Group: 产品需求和PRD

## 核心目标
本小组是 vibe coding 流程中「产品需求与 PRD」的专项讨论组。

## 角色职责
- **Sundar Pichai (Pichai)**: 首席产品经理
  - 接收原始需求，进行深度对齐。
  - 遵循 MECE 原则生成 PRD（产品需求文档）。
  - 组织下游研发环节（如 Claude Code 实现）及验收环节（如 fty-nb 验证）。

## 流程准则 (Vibe Flow)
1. **需求输入**: 接收来自老板 (Jo) 或其他成员的原始想法。
2. **对齐与 PRD**: Pichai 编写 PRD 并获取确认。
3. **开发组织**: Pichai 发起实现任务。
4. **验收闭环**: 完成后进行 GUI 自动化验收，确保质量。
